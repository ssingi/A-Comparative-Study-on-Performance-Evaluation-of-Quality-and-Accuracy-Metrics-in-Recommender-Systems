### 예제 2: 영화 감상

4시간 영화를 봤는데 친구가 "30초로 요약해줄 수 있어?"라고 물어봅니다.

**4시간 영화 전체:**
```
도입 부분 (30분)
스토리 전개 (60분)
↓↓ 클라이맥스 (15분) ← 가장 중요!
스토리 전개 (75분)

30초 요약:
→ "주인공이 악당을 물리치고 세상을 구했어" (클라이맥스만!)

**전체 정보는 1% 남지만, 가장 중요한 내용 100%가 들어있어요!**

이것도 **TruncatedSVD의 개념**입니다.

## 🎯 핵심만 고르기

### SNS에 올리는 사진

인스타그램에 사진을 올릴 때:

**원본 사진의 특징들:**
```
1. 주인공의 얼굴 ← 가장 중요! (95%)
2. 배경 ← 조금 중요 (4%)
3. 조명 ← 거의 중요하지 않음 (0.8%)
4. 날씨 흔적 ← 잡음 (0.2%)
```

**필터링 후:**
```
1. 주인공의 얼굴만 강조
2, 3, 4는 제거 또는 약화
```

**결과:**
- 용량: 90% 감소 (5MB → 0.5MB)
- 속도: 업로드 10배 빨라짐
- 품질: 그대로 (사진은 여전히 예쁨)

→ **이것이 TruncatedSVD 개념입니다!**

### 흐릿한 사진을 선명하게

카메라로 찍은 사진이 흐릿해요. 왜일까요?

**흐릿한 사진 = 여러 정보가 섞여있음**
```
사진에 들어있는 정보:
- 주인공 (70%) ← 중요
- 배경 (20%) ← 중요
- 카메라 떨림 (5%) ← 잡음
- 먼지 흔적 (3%) ← 잡음
- 색감 오류 (2%) ← 잡음
```

**포토샵으로 편집:**
```
1. 카메라 떨림 제거
2. 먼지 흔적 제거
3. 색감 오류 수정

결과: 선명한 사진!
```

**무엇을 한 걸까?**

→ 중요한 정보는 유지, 불필요한 정보는 제거!

**이것이 TruncatedSVD입니다!**

## 🎬 영화 추천으로 이해하기

### 넷플릭스의 추천 시스템

넷플릭스가 "당신이 좋아할 만한 영화"를 추천해줍니다. 어떻게 알까요?

#### 방법 1: 나이브한 방법 (TruncatedSVD 없이)

**모든 사용자 × 모든 영화를 저장**
```
1000만 명 사용자 × 10만 개 영화
= 10억 개 데이터

컴퓨터 메모리: 엄청 많이 필요 (수십 TB)
계산 시간: 수시간 (너무 느림)
```

#### 방법 2: 똑똑한 방법 (TruncatedSVD 사용)

**숨겨진 특징만 저장**
```
사용자: [0.8(액션), 0.2(드라마), 0.5(공포), ... (200개)]
영화: [0.9(액션), 0.1(드라마), 0.7(공포), ... (200개)]

200개 숫자만으로 모든 취향 표현 가능!
```

**효과:**
```
메모리: 100배 감소!
속도: 100배 빨라짐!
추천 품질: 오히려 더 좋음!
```

**왜?**
- 1000만 × 10만 = 10억 개 데이터
- 1000만 × 200 + 200 × 10만 = 약 3000만 개 데이터
- 97% 메모리 절감!

## 🧠 생각해볼 점

### Q1: 모든 정보를 다 유지하면 더 좋지 않을까?

**A:** 아니요. 이유:

```
생일 파티 사진에서:
- 주인공 얼굴 (95% 중요)
- 배경 풍선 (4% 중요)
- 사진 노이즈 (1% 해로움)

1%의 노이즈가 전체 사진을 흐릿하게 만듦.

노이즈 제거 후:
→ 사진이 더 선명하고 깔끔!
```

### Q2: 어떻게 중요한 정보와 불필요한 정보를 구분할까?

**A:** 수학적 방법을 사용합니다.

```
간단 버전:
1. 모든 데이터를 정렬
2. 크기가 큰 것부터 순서대로 배열
3. 앞에서부터 필요한 만큼만 선택

예: 우리 반의 키
[185, 178, 175, 172, 170, 168, ...]
앞에서부터 선택 = 키 큰 학생부터 선택
```

### Q3: 그럼 몇 개를 선택해야 할까?

**A:** 필요한 정보량에 따라 다릅니다.

```
예: 영화 추천
- 10개 특징: 빠르지만 정확도 낮음
- 50개 특징: 괜찮음
- 200개 특징: 좋음 (당신의 코드)
- 500개 특징: 매우 좋지만 느림
- 1000개 특징: 원본과 거의 같음 (의미 없음)

보통: 50~500개 사이 선택
```

### Q4: 실제로 사용되나요?

**A:** 네, 아주 광범위하게!

```
✓ 넷플릭스: 영화 추천
✓ 스포티파이: 음악 추천
✓ 유튜브: 영상 추천

## 🎓 결론: 고등학생도 이해할 수 있는 핵심

### SVD = 요약하는 방법

```
마치 책의 목차처럼, 핵심만 모아놓는 것입니다.

📖 전체 책
  - 400페이지
  - 시간이 오래 걸림

📑 목차만 보기
  - 2페이지
  - 빠르게 내용 파악
  - 필요하면 해당 부분만 읽음
```

### TruncatedSVD = 더 똑똑한 요약

```
마치 AI 요약처럼, 가장 중요한 정보만 골라내는 것입니다.

📰 전체 기사 (복잡함)
📝 AI 요약 (핵심만, 이해하기 쉬움)
```

### 3가지 효과

```
1️⃣ 빨라짐
   - 처리 시간 100배 단축
   - 결과 나옴: 1시간 → 1분

2️⃣ 작아짐
   - 메모리 99% 감소
   - USB에 들어가는 데이터

3️⃣ 더 좋아짐
   - 잡음 제거로 더 정확해짐
   - 예: 흐릿한 사진 → 선명한 사진
```

### 언제 사용할까?

```
"너무 큰 데이터라서 처리가 안 될 때"

예:
- 넷플릭스: 1000만 명 × 10만 영화 = 너무 큼!
  → TruncatedSVD로 축소

- 유튜브: 수억 개 영상 × 수억 명 사용자
  → 너무 커서 TruncatedSVD 필수

- 당신의 학교 급식 시스템: 
  → 너무 작아서 필요 없음 (모두 저장 가능)
```
